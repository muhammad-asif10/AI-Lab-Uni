üìÖ Week 14 ‚Äî Model Representation & Cost Function

üéØ Main idea: Predict + Measure error
What students should remember
1) Model makes predictions
2) Cost tells how wrong predictions are
3) Lower cost = better model
One-line formula
Prediction then Error
(First we predict, then we check how wrong we are.)
Real-life analogy: Throwing a ball at a target üéØ
‚Ä¢ Model = your throwing technique
‚Ä¢ Prediction = where ball lands
‚Ä¢ Cost/Error = distance from target
‚Ä¢ Training = improving technique to reduce distance

üß± Model Representation (Very Simple)

üß† Model: Input ‚Üí Rule ‚Üí Output

In AI, a model is just a rule that converts input into output. For linear regression, the rule is:

Model equation: y = w √ó x + b
‚Ä¢ x = input (feature) like house size
‚Ä¢ w = how strongly x affects y (rate)
‚Ä¢ b = starting value (base value)
‚Ä¢ y = predicted output
Example (rent)
If w = 2000 and b = 5000, and x = 3 (3 marla or 3 unit):
y = 2000√ó3 + 5000 = 11000

üêç Model in Python (OOP) ‚Äî Extra Commented

LinearModel class with line-by-line comments
# =========================
# LINEAR MODEL (Week 14)
# Equation: y = w*x + b
# =========================

class LinearModel:
    def __init__(self, w, b):
        # Store model parameters inside the object
        # w = weight (slope)
        # b = bias (intercept)
        self.w = w
        self.b = b

    def predict(self, x):
        # This function makes a prediction for ONE input x
        # Step 1: multiply x with weight w
        wx = self.w * x

        # Step 2: add bias b
        y = wx + self.b

        # Step 3: return predicted output y
        return y

# ----- Example usage -----
model = LinearModel(w=2, b=1)       # create model
result = model.predict(5)           # predict for x = 5
print("Prediction:", result)        # expected: 11
Simple check
If w increases, the line becomes steeper (bigger change).
If b increases, the whole line shifts upward.

üéØ Cost Function (Why we need it) ‚Äî Super Simple

üìâ Cost: tells how wrong we are

A model can be right or wrong. We need a number that tells us how wrong. That number is called cost (or loss).

Error for one point: error = predicted ‚àí actual
If predicted is far from actual ‚Üí error is big ‚Üí model is not good.
Marks example
Actual = 80, Predicted = 60 ‚Üí error = -20 (distance is 20).
We don‚Äôt want negative/positive confusion, so we use squared error in MSE.

üßÆ MSE (Simple Python) ‚Äî ‚ÄúLoop style‚Äù

üß† MSE: average of squared errors
MSE idea in plain words:
1) For every data point, find error
2) Square the error (make it positive and bigger for big mistakes)
3) Add all squared errors
4) Divide by total points (average)
MSE using simple Python lists
# =========================
# MSE (Week 14) - Simple Python
# =========================

def mean_squared_error(y_actual, y_pred):
    # n = how many data points
    n = len(y_actual)

    # total will store sum of squared errors
    total = 0

    # loop through each point
    for i in range(n):
        # error = predicted - actual
        error = y_pred[i] - y_actual[i]

        # squared error
        squared_error = error ** 2

        # add to total
        total += squared_error

    # average (mean)
    mse = total / n
    return mse

# Example
y_actual = [3, 5, 7]
y_pred   = [2.5, 5.5, 6]
print("MSE:", mean_squared_error(y_actual, y_pred))
Teacher tip
Ask students: ‚ÄúWhy do we divide by n?‚Äù ‚Üí because we want average mistake, not total mistake.

üìä MSE (Data Science Style) ‚Äî NumPy

‚ö° Fast: vectorized calculation

In data science, we usually use NumPy arrays for fast calculations. Same MSE formula, just written shorter and faster.

One-line MSE: np.mean((y_pred - y_actual) ** 2)
Here y_pred - y_actual happens for all values at once (vectorized).
MSE using NumPy (DS style)
import numpy as np

# Convert lists into NumPy arrays
y_actual = np.array([3, 5, 7], dtype=float)
y_pred   = np.array([2.5, 5.5, 6], dtype=float)

# Data science style MSE (vectorized)
mse = np.mean((y_pred - y_actual) ** 2)

print("MSE (NumPy):", mse)
Simple meaning
NumPy does the same loop internally, but much faster.

üîó Connect Model + MSE (Full Pipeline)

üß™ Flow: x ‚Üí predict ‚Üí MSE
Step-by-step:
1) Create model (choose w,b)
2) Predict outputs for all x values
3) Compare with actual y values using MSE
Pipeline code (very clear)
# =========================
# Pipeline: Model -> Predictions -> MSE
# =========================

class LinearModel:
    def __init__(self, w, b):
        self.w = w
        self.b = b

    def predict(self, x):
        return self.w * x + self.b

def mean_squared_error(y_actual, y_pred):
    n = len(y_actual)
    total = 0
    for i in range(n):
        total += (y_pred[i] - y_actual[i]) ** 2
    return total / n

# Dataset
x = [1, 2, 3, 4, 5]
y_actual = [3, 5, 7, 9, 11]   # true: y = 2x + 1

# Try any model
model = LinearModel(w=1.4, b=1.2)

# Make predictions
y_pred = []
for xi in x:
    y_pred.append(model.predict(xi))

# Calculate cost
cost = mean_squared_error(y_actual, y_pred)

print("Predictions:", y_pred)
print("Cost (MSE):", cost)
Key learning
If cost is high, we must change w and b. Week 15 shows how to do it automatically.

üìÖ Week 15 ‚Äî Gradient Descent (Training)

üöÄ Training: reduce MSE automatically

Gradient Descent is the learning method that finds better w and b values. It repeats small updates until MSE becomes small.

Very simple
Gradient Descent = ‚Äúreduce cost step by step‚Äù
Analogy
Like going down a hill to reach the lowest point (minimum cost).

üß≠ Gradient Descent Concept (Step-by-step)

üß† Idea: small improvements again and again
What happens in every epoch:
1) Use current w,b to make predictions
2) Calculate MSE (cost)
3) Find direction to reduce cost (gradients dw, db)
4) Update w,b with a small step (learning rate Œ±)
5) Repeat
TermMeaning (simple)
EpochOne training round (one update step)
Learning rate (Œ±)How big step you take
Gradient (dw, db)Direction that reduces cost

üìå Update Rule (Easy)

üìå Update: new = old ‚àí (step)
Update equations:
w = w ‚àí Œ± √ó dw
b = b ‚àí Œ± √ó db

Meaning: If dw is positive, w decreases a bit. If dw is negative, w increases a bit.
Learning rate warning
Too big Œ± ‚Üí may overshoot and cost can increase.
Too small Œ± ‚Üí training works but becomes slow.

üêç Gradient Descent Code (Extra Commented)

Beginner-friendly GD code
# =========================
# GRADIENT DESCENT (Week 15)
# Model: y = w*x + b
# Cost:  MSE
# =========================

def predict_all(x, w, b):
    # Predict y for ALL x values
    y_pred = []
    for xi in x:
        y_pred.append(w * xi + b)
    return y_pred

def mean_squared_error(y_actual, y_pred):
    # Compute MSE using simple loop
    n = len(y_actual)
    total = 0
    for i in range(n):
        total += (y_pred[i] - y_actual[i]) ** 2
    return total / n

def gradient_descent(x, y, w, b, alpha, epochs):
    # n = number of points
    n = len(x)

    for epoch in range(epochs):
        # 1) Predictions with current parameters
        y_pred = predict_all(x, w, b)

        # 2) Compute gradients (dw and db)
        # Start from 0
        dw = 0
        db = 0

        for i in range(n):
            # error for this point
            error = y_pred[i] - y[i]

            # contribution to gradient of w and b
            dw += error * x[i]
            db += error

        # 3) Average gradients and scale for MSE derivative
        dw = (2 / n) * dw
        db = (2 / n) * db

        # 4) Update parameters in the direction that reduces MSE
        w = w - alpha * dw
        b = b - alpha * db

        # 5) Print progress
        if epoch % 10 == 0:
            cost = mean_squared_error(y, predict_all(x, w, b))
            print("Epoch", epoch, "| w =", round(w,4), "| b =", round(b,4), "| MSE =", round(cost,4))

    return w, b

# ----- Example dataset (true: y = 2x + 1) -----
x = [1, 2, 3, 4, 5]
y = [3, 5, 7, 9, 11]

# Start with wrong w,b
w0 = 0.0
b0 = 0.0

# Train
w_final, b_final = gradient_descent(x, y, w0, b0, alpha=0.05, epochs=120)

print("Final:", "w =", round(w_final,4), "b =", round(b_final,4))
What students should see
MSE decreases with epochs. Final w,b become close to 2 and 1.

üìà Visual: MSE Decreasing (Plot)

This plot shows how MSE goes down during training (Gradient Descent).

Preview included for teaching. Students can generate using the code below.

Matplotlib code: plot MSE vs epochs
import numpy as np
import matplotlib.pyplot as plt

# dataset
x = np.array([1,2,3,4,5], dtype=float)
y = np.array([3,5,7,9,11], dtype=float)   # true: y=2x+1

# start values
w, b = 0.0, 0.0
alpha = 0.05
epochs = 60
costs = []

n = len(x)

for epoch in range(epochs):
    # predictions
    y_pred = w*x + b

    # cost
    cost = np.mean((y_pred - y)**2)
    costs.append(cost)

    # gradients
    dw = (2/n) * np.sum((y_pred - y) * x)
    db = (2/n) * np.sum((y_pred - y))

    # update
    w = w - alpha*dw
    b = b - alpha*db

# plot
plt.figure(figsize=(7,4))
plt.plot(range(1, epochs+1), costs)
plt.title("MSE decreasing during Gradient Descent")
plt.xlabel("Epoch")
plt.ylabel("MSE")
plt.tight_layout()
plt.show()
